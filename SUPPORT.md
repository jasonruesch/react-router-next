# Getting help with react-router-next

This is a community-maintained open-source project — support is best-effort. Here's where to go for what.

## Documentation first

Most questions are answered in the docs:

- [Root README](README.md) — repo layout and how to work in the monorepo
- [Package README](packages/react-router-next/README.md) — installation, conventions, all public APIs

## Asking a usage question

For "how do I…?" questions, start a [GitHub Discussion](https://github.com/evolonix/react-router-next/discussions). Discussions are searchable, so future users with the same question benefit from the answer.

When asking, please include:

- A minimal code sample showing what you tried
- Versions: `@evolonix/react-router-next`, `react-router`, `react`, `vite`, Node
- What you expected vs. what happened

## Reporting a bug

If you've found a reproducible bug, file an [issue with the bug report template](https://github.com/evolonix/react-router-next/issues/new?template=bug_report.yml). A minimal reproduction (CodeSandbox, StackBlitz, or a small repo) is the single biggest factor in how fast a bug gets fixed.

## Requesting a feature

Open a [feature request](https://github.com/evolonix/react-router-next/issues/new?template=feature_request.yml) and describe the use case before the proposed implementation.

## Reporting a security vulnerability

**Do not** open a public issue. See [SECURITY.md](.github/SECURITY.md) for the private reporting process.

## Response times

This project is maintained in spare time. Issues and discussions are typically triaged within a week, but there's no SLA. PRs that include a clear reproduction or test case land faster than ones that don't.
